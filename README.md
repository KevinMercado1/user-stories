# User Stories Project 

A simple repository to manage and document project requirements using the **User Stories** methodology.

---

## 🛠 Features
* **Clear Roles:** Definition of different types of users.
* **Acceptance Criteria:** Specific conditions for each story to be "Done".
* **INVEST Principle:** Ensuring every story is independent and valuable.

## Structure
* `Backlog/`: List of pending features.
* `Completed/`: Stories already implemented.
* `Assets/`: Images and diagrams.
